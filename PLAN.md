## Tech stack (optimized for speed + Claude chat workflow)

- **Frontend:** vanilla HTML / CSS / JS (no build step) → instant GitHub Pages deploy
- **Data:** JSON files (`data/talent.json`, `data/startups.json`) — synthetic, Utah-specific
- **Matching:** weighted scoring in `match.js` (deterministic, fast, explainable)
- **LLM layer:** Claude API for natural-language "Why matched?" explanations (fallback to templates if no key)
- **Hosting:** GitHub Pages (free, push-to-deploy)
- **Branding:** Utah-themed palette — mountain navy `#1B365D`, sandstone `#C8A165`, snow white, U-of-U red accent `#CC0000`

Why this stack: every file can be drafted in Claude chat and pasted in. No npm, no build, no deploy headaches.

## Person B — Matching Engine, Data & Integration (owns 30% match + 20% integration)

**Goal:** the brain. Synthetic data, scoring, explanations, integrations.

Tasks:

1. **Synthetic Utah dataset** (`data/talent.json` + `data/startups.json`)
   - 50+ talent profiles: students/alumni from **BYU, U of U, USU**, plus ex-operators from Qualtrics, Pluralsight, Domo, Recursion, Pattern, Weave, Lucid, Podium
   - 25+ startups across all 8 sectors, mix of university spinouts + bootstrapped
   - Use real Utah cities: Salt Lake, Provo, Lehi, Ogden, Logan, Park City
   - **Claude prompt:** "Generate 50 realistic Utah talent profiles as JSON matching this schema: …"

2. **Matching algorithm** (`match.js`)
   ```
   score = 0.25*sector + 0.25*skills + 0.15*stage
         + 0.10*availability + 0.10*risk + 0.15*mission
   ```
   - `sector`: exact match = 1.0; adjacent (e.g. AI ↔ software) = 0.6
   - `skills`: Jaccard overlap between talent skills and startup needs
   - `stage`: 1 if startup stage in talent.stage_pref, else 0
   - `availability`: matrix lookup (full-time × CEO need = 1.0, etc.)
   - `risk`: lower delta = higher score (university spinout → needs higher risk tolerance)
   - `mission`: cosine similarity of mission text, OR Claude-rated 0–1
   - Returns `{ score, breakdown, top_reasons[] }`

3. **Explainability** (`explain.js`)
   - Template-based first ("Strong sector fit (advanced manufacturing). Skills overlap 4/5. Talent prefers pre-seed; HelioMet is pre-seed.")
   - If Claude API key available → call Claude to rewrite into 3 natural-language bullets with a friendly Utah voice

4. **Integrations** (the 20% nobody else will nail)
   - **Affinity webhook mock** (`integrations/affinity.js`): POST to `https://webhook.site/...` with the matched pair, show toast "Sent to Affinity CRM ✓"
   - **Squarespace embed**: a `<iframe>` snippet + `embed.html` that the Nucleus team can paste into their Squarespace block — document this in README
   - Add an **"Export CSV"** button for Affinity import (talent_id, startup_id, score, top_reasons)

5. **Bonus (only if ahead at 12:00):**
   - Ecosystem mini-map: D3 force graph showing connections (talent ↔ school ↔ startup)
   - "80% fit — close the gap" upskilling card on talent profile
